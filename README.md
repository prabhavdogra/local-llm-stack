# Local LLM Stack

Run local LLMs with a unified API gateway, chat UI, and monitoring — all from a single `models.yaml`.

## Architecture

```
┌─── Mac (edit code here) ────────────────────────────────────┐
│  make sync / make remote-up   →   rsync over SSH            │
└─────────────────────────────────────────────────────────────┘
         │
         ▼
┌─── DGX Spark (run inference) ───────────────────────────────┐
│  Docker Compose:                                            │
│    ├── vLLM       (:8000)  GPU inference                    │
│    ├── LiteLLM    (:4000)  API gateway → vLLM              │
│    ├── Open WebUI (:3001)  Chat UI → LiteLLM               │
│    ├── Prometheus (:9090)  Scrapes /metrics                 │
│    └── Grafana    (:3000)  Dashboards                       │
└─────────────────────────────────────────────────────────────┘
         │
         ▼ (optional)
┌─── ngrok ───────────────────────────────────────────────────┐
│  Public HTTPS tunnel → LiteLLM (for Cursor IDE, etc.)       │
└─────────────────────────────────────────────────────────────┘

macOS-only alternative (no DGX):
  └── MLX-LM Server (:8800)  →  LiteLLM via host.docker.internal
```

All inference flows through LiteLLM. Swapping the backend = updating `models.yaml`.

## Quick Start (DGX Spark)

From your Mac, with SSH access to `spark-2393.local`:

```bash
# 1. Sync code to DGX
make sync

# 2. First-time setup — create .env + generate secrets on DGX
make remote-config

# 3. Start the full stack on DGX
make remote-up

# 4. Access services from your Mac (port-forward)
ssh -L 3001:127.0.0.1:3001 -L 4000:127.0.0.1:4000 -L 3000:127.0.0.1:3000 prabhav@spark-2393.local
```

Then open http://localhost:3001 (WebUI) or hit http://localhost:4000 (API) on your Mac.

## Setup

### Environment Variables

`make config` (or `make up` on first run) copies `.env.example` → `.env` and
auto-generates `LITELLM_MASTER_KEY` and `GRAFANA_ADMIN_PASSWORD`.

```bash
make check-env   # report what's set vs missing
```

| Variable | When required |
|----------|--------------|
| `LITELLM_MASTER_KEY` | Always (auto-generated by `make config`) |
| `GRAFANA_ADMIN_PASSWORD` | Always (auto-generated by `make config`) |
| `HF_TOKEN` | Gated/private Hugging Face models (e.g. `meta-llama/*`) |
| `NGROK_AUTHTOKEN`, `NGROK_DOMAIN` | Only for `make up-ngrok` |

### MLX backend (macOS only)

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
make up BACKEND=mlx
```

## DGX Spark Deployment

### Prerequisites

- SSH access: `ssh prabhav@spark-2393.local`
- Docker + NVIDIA Container Toolkit installed:

```bash
docker run --rm --gpus all nvidia/cuda:12.0.0-base-ubuntu22.04 nvidia-smi
```

### Sync & Run (from Mac)

Edit code on the Mac, run inference on the DGX:

```bash
make sync              # rsync project → DGX (skips .env — secrets stay remote)
make sync-dry          # preview what would sync
make remote-config     # first time: create .env + secrets on DGX
make remote-up         # sync + start stack on DGX
make ssh-dgx           # SSH into the project dir on DGX
```

Defaults (override via env or command line):
- `DGX_HOST=prabhav@spark-2393.local`
- `DGX_PATH=~/Desktop/repositories/local-llm-stack`

```bash
make sync DGX_HOST=user@other-host DGX_PATH=~/my-stack
```

### Or run directly on the DGX

```bash
cd ~/Desktop/repositories/local-llm-stack
make config        # creates .env + generates secrets
make check-env     # verify everything needed is set
make up            # start vLLM + LiteLLM + WebUI + Grafana
make logs s=vllm   # watch model download (first run)
make health        # wait for all services healthy
make test          # smoke-test the API
```

### Access from Mac

Services bind to `127.0.0.1` on the DGX. Forward ports via SSH:

```bash
ssh -L 3001:127.0.0.1:3001 \
    -L 4000:127.0.0.1:4000 \
    -L 3000:127.0.0.1:3000 \
    prabhav@spark-2393.local
```

Or use ngrok for public HTTPS (e.g. Cursor IDE remote model):

```bash
# Set NGROK_AUTHTOKEN and NGROK_DOMAIN in .env on the DGX, then:
make up-ngrok
```

### Model Selection

Default: `Qwen/Qwen3-Coder-Next-FP8` (fits DGX Spark 128 GB unified memory).

```bash
# Edit .env on the DGX:
VLLM_MODEL=Qwen/Qwen2.5-Coder-32B-Instruct
make restart
```

Available presets in `.env.example`:
| Model | Notes |
|-------|-------|
| `Qwen/Qwen3-Coder-Next-FP8` | Default, FP8 quantized |
| `Qwen/Qwen3-Coder-Next` | BF16, needs >160 GB |
| `bullpoint/Qwen3-Coder-Next-AWQ-4bit` | AWQ INT4, ~40 GB |
| `Qwen/Qwen2.5-Coder-32B-Instruct` | Smaller, fast |
| `Qwen/Qwen3-8B` | Lightweight |

## Adding a Model

**MLX backend** — add to `models.yaml`:

```yaml
  - model_name: deepseek-coder-v2:16b
    litellm_params:
      model: openai/mlx-community/DeepSeek-Coder-V2-Instruct-4bit
      api_key: none
      api_base: http://host.docker.internal:8800/v1
    model_info:
      id: deepseek-coder-v2-16b
      description: "MoE 16B code model from DeepSeek."
      tasks: [code]
      params_b: 16
      active_b: 2.4
      arch: moe
      quant: 4bit
      context: 131072
      vram_gb: 9
```

**NVIDIA backend** — change `VLLM_MODEL` in `.env` and `make restart`.

Then reload LiteLLM:

```bash
make restart-litellm
```

## Monitoring

Open Grafana at http://localhost:3000. Log in as `admin` with the
`GRAFANA_ADMIN_PASSWORD` from `.env`. Anonymous access is disabled.

The provisioned dashboard ("LLM Inference Metrics") shows:
- **Overview** — requests in-flight, total, failed, LiteLLM memory
- **Throughput** — tokens/sec, request rate by model
- **Latency** — end-to-end request latency (p50/p95), LLM API latency
- **Tokens & Models** — cumulative tokens by model, latency per output token

## Make Targets

### Stack

```
make up                  Start everything (creates .env on first run)
make down                Stop all services
make restart             Restart everything
make logs                Tail all logs (make logs s=litellm for one service)
make status              Show running services + loaded models
make health              Probe all health endpoints
make test                Smoke-test the LiteLLM API
make clean               Stop + delete all volumes (destructive)
```

### Configuration

```
make config              Create .env from .env.example + generate secrets
make config FORCE=1      Reset .env and regenerate all secrets
make check-env           Report required / missing .env vars
```

### DGX Sync

```
make sync                Rsync project to DGX (skips .env)
make sync-dry            Preview rsync (no changes)
make ssh-dgx             SSH into project dir on DGX
make remote-config       Create .env on DGX (first time)
make remote-up           Sync + start stack on DGX
```

### Models

```
make pull MODEL=...      Pre-download a model from HuggingFace
make models              List downloaded models
make restart-litellm     Reload models.yaml changes
```

### MLX (macOS only)

```
make mlx-start           Start MLX server
make mlx-stop            Stop MLX server
make mlx-logs            Tail MLX server log
```

## Swapping Components

| Component | Swap for | What to change |
|-----------|----------|----------------|
| MLX-LM | vLLM / Ollama | Update `models.yaml` api_base + backend service |
| LiteLLM | Nginx / Traefik | Point Open WebUI at new proxy; update Makefile |
| Open WebUI | LobeChat / LibreChat | Replace the `open-webui` service in compose |
| Grafana | Netdata / Uptime Kuma | Replace the `grafana` service, keep Prometheus |
| Prometheus | VictoriaMetrics | Drop-in replacement, same scrape config |

The key abstraction: **all consumers talk to LiteLLM, never directly to the backend**.
